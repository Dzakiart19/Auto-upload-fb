Kamu adalah AI developer Replit yang bertugas memperbaiki koneksi server Mastra agar webhook Telegram bisa diakses publik tanpa error â€œFailed to resolve hostâ€.

Tugasmu sekarang:
1. Pastikan server Express Mastra berjalan terus-menerus di port 5000.
2. Pastikan ada endpoint `/status` dan `/webhooks/telegram/action` yang benar-benar merespon â€œokâ€.
3. Tampilkan URL publik aktif di console log, misalnya `https://<projectname>.<username>.repl.co` atau `.replit.dev`.
4. Pastikan URL yang digunakan untuk webhook Telegram adalah URL publik yang sedang aktif, bukan URL mati.
5. Jika proyek sedang berhenti saat tidak ada request, aktifkan keep-alive (gunakan ping loop atau UptimeRobot-friendly endpoint).
6. Setelah server aktif, jalankan perintah berikut dengan token dari `.env`:

https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/setWebhook?url=https://<url-publik-aktif>/webhooks/telegram/action

7. Verifikasi webhook dengan:

https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/getWebhookInfo

8. Pastikan respons `{ "ok": true }` muncul.
9. Jika `.replit.dev` tidak aktif, buat fallback otomatis ke `.repl.co` (pakai `process.env.REPL_SLUG` dan `process.env.REPL_OWNER` untuk membangun URL-nya).
10. Beritahu saya URL publik akhir yang bisa diakses browser agar saya bisa cek `/status`.

Gunakan bahasa sederhana di log agar mudah dibaca. Jangan gunakan fitur deployment berbayar, cukup mode gratis.
